# <a name="source-environment-expectations-for-office-365-us-government"></a>Erwartungen an die Quellumgebung für Office 365 US Government

Der FastTrack Center-Vorteil stellt Anweisungen zum Einrichten von Integrationsstufen für Ihre Quellumgebung bereit. (Beispiel: Ihre Quellumgebung weist bereits Dienste auf, die Sie in Office 365 verlagern möchten).
  
> [!NOTE]
> Falls eine Integration erforderlich ist, muss sich Ihre Quellumgebung auf der Mindeststufe für die jeweilige Anwendung befinden. 
  
In der folgenden Tabelle werden die Anforderungen an die vorhandene Quellumgebung für das Onboarding beschrieben.

|**Aktivität**|**Anforderung an die Quellumgebung**|
|:-----|:-----|
|Haupt-Onboarding | Active Directory-Gesamtstrukturen mit Windows Server 2003 oder höher als Funktionsgesamtstrukturebene mit der folgenden Gesamtstrukturkonfiguration:  <br/>      Einzelne Active Directory-Gesamtstruktur  <br/>    Eine einzelne Active Directory-Kontogesamtstruktur und Ressourcen-Gesamtstrukturtopologien (Exchange und/oder Lync 2010, Lync 2013 oder Skype for Business)  <br/>  Mehrere Active Directory-Kontogesamtstrukturen und Ressourcen-Gesamtstrukturtopologien (Exchange und/oder Lync 2010, Lync 2013 oder Skype for Business)  <br/>  Mehrere Active Directory-Kontogesamtstrukturen, wobei eine der Gesamtstrukturen eine Gesamtstruktur mit einem zentralen Active Directory-Konto ist, die Exchange und/oder Lync 2010, Lync 2013 oder Skype for Business umfasst.  <br/>  Mehrere Active Directory-Konto-Gesamtstrukturen mit einer jeweils eigenen Exchange-Organisation <br/> <br/> **Hinweis**  *Für Active Directory-Szenarios mit mehreren Gesamtstrukturen müssen, wenn Lync 2010, Lync 2013 oder Skype for Business bereitgestellt wird, diese in der gleichen Active Directory-Gesamtstruktur wie Exchange bereitgestellt werden.*   <br/>  <br/>      **Hinweis**  *Bei der Implementierung mehrerer ActiveDirectory-Gesamtstrukturen mit mehreren Exchange-Organisationen in eine Exchange-Multihybridkonfiguration werden freigegebene Namespaces für Benutzerprinzipalnamen (UPN) zwischen Quellgesamtstrukturen nicht unterstützt. Primäre SMTP-Namespaces zwischen Exchange-Organisationen sollten ebenfalls getrennt werden. Weitere Informationen finden Sie unter [Hybrid-Bereitstellungen mit mehreren Active Directory-Gesamtstrukturen](https://go.microsoft.com/fwlink/?linkid=845444).*     <br/>   <br/>   **Hinweis**  *Für alle Konfigurationen mit mehreren Gesamtstrukturen liegt die AD FS-Bereitstellung außerhalb des FastTrack Center-Vorteils.*           |
|Exchange Online-Onboarding | Ihre Umgebung muss eine der folgenden Mindeststufen aufweisen:  <br/>  Einzelne oder mehrere Exchange-Organisationen mit Exchange Server 2003 oder höher.  <br/>  Einzelne IMAP-fähige E-Mail-Umgebung.  <br/>  Eine einzelne G Suite-Umgebung (nur Gmail, Kontakte und Kalender).             |
|Sharepoint Online- und OneDrive for Business-Onboarding  | **SharePoint-Hybridlösung**  <br/>  Die SharePoint-Hybridkonfiguration umfasst das Konfigurieren der Hybridsuche, von Websites, Taxonomie, Inhaltstypen, OneDrive for Business, ein erweitertes App-Startfeld und Extranet-Websites und Self-Service Site Creation, die von einer lokalen Umgebung mit einer SharePoint Online-Umgebung mit einem Ziel verbunden werden. Zum Aktivieren der SharePoint-Hybridlösung benötigen Sie eine der folgenden lokalen SharePoint Server-Umgebungen: <br/> <br/> **_SharePoint Server 2013_*_ <br/> Öffentliches Update (PU) vom Juni 2017 für Hybridinhaltstypen. Weitere Informationen finden Sie unter [Konfigurieren von hybriden SharePoint Taxonomie und Hybridinhaltstypen.](https://go.microsoft.com/fwlink/?linkid=853547)  <br/>Öffentliches Update vom März 2017 für die Self-Service Site Creation-Konfiguration. Die hybride Self-Service Site Creation wird nur mit SharePoint Server 2013 unterstützt. Weitere Informationen finden Sie unter [Hybrid Self-Service Site Creation](https://go.microsoft.com/fwlink/?linkid=846015).  <br/>Öffentliches Update vom November 2016 mit Hybridtaxonomie. Weitere Informationen finden Sie unter [Planen der hybriden SharePoint Taxonomie.](https://go.microsoft.com/fwlink/?linkid=844867)  <br/>Öffentliches Update vom Juli 2016 für alle anderen Hybridkonfigurationsszenarien.  <br/> <br/> _* Hinweis SharePoint***Server 2013-Hybridszenarien nur mit SharePoint Server 2013 unterstützt werden. Diese Szenarien werden in SharePoint Foundation 2013 nicht unterstützt.*    <br/>   <br/>    ***SharePoint Server 2016** _  <br/>  Öffentliches Juni 2017-Update für hybride Inhaltstypen. Weitere Informationen finden Sie unter [Konfigurieren der hybriden SharePoint-Taxonomie und hybriden Inhaltstypen](https://go.microsoft.com/fwlink/?linkid=853547).<br/>  Öffentliches November 2016-Update (Feature Pack 1) mit Hybridtaxonomie. Weitere Informationen finden Sie unter [Planen der SharePoint-Hybridtaxonomie und hybriden Inhaltstypen](https://go.microsoft.com/fwlink/?linkid=844867).<br/>  Öffentliches Juli 2016-Update und höher für alle anderen Hybridkonfigurationsszenarios.  <br/><br/> _ *   *Hinweis: Das Upgrade von lokalen SharePoint-Umgebungen auf SharePoint Server 2013 oder SharePoint Server 2016 wird nicht vom FastTrack Center-Vorteil bereitgestellt. Weitere Informationen finden Sie unter [Minimale öffentliche Updateebenen für SharePoint Hybridfeatures.](https://go.microsoft.com/fwlink/?linkid=853548)*             |
|Skype for Business Online-Onboarding  | **Netzwerkbewertung**  <br/>  Überprüfungen hinsichtlich der Ports und Endpunkte  <br/>  Überprüfungen hinsichtlich der Verbindungsqualität  <br/>  Schätzungen hinsichtlich der Bandbreite  <br/><br/>  **Lync Hybrid**  <br/>  Einzelne lokale Active Directory-Gesamtstruktur.  <br/>  Eine Lync 2010-Serverumgebung mit Lync 2013-Verwaltungstools oder Skype for Business 2015-Verwaltungstools und Lync 2010-Edge-Serverrolle.  <br/>  Lync 2013-Serverumgebung und Lync 2013-Edge-Serverrolle.  <br/><br/>  **Skype for Business Online Hybrid**  <br/>  Einzelne lokale Active Directory-Gesamtstruktur.  <br/>  Einzelne Active Directory-Konto-Gesamtstruktur und Ressourcen-Gesamtstrukturtopologien (Exchange und/oder Skype for Business)  <br/>  Mehrere Active Directory-Kontogesamtstrukturen, wobei eine der Gesamtstrukturen eine Gesamtstruktur mit einem zentralen Active Directory-Konto mit Exchange und/oder Skype for Business ist.  <br/>  Skype for Business Server 2015-Umgebung mit Skype for Business-Edge-Serverrolle.  <br/><br/> **Hinweis**  *Dieser zusätzliche Dienst dient zum Konfigurieren und Überprüfen der Aufgaben für geteilte Domänen (Hybrid) und umfasst keine Einführung in die lokalen Komponenten (z. B. Lync 2013-Verwaltungstools oder Lync 2013/Skype for Business Online-Server oder Lync 2013, Lync 2010 oder Skype for Business-Edge-Server).*    <br/><br/>        **Gerät für den Konferenzraum**  <br/>  Erstellung von Onlinekonten, die für die unterstützten Konferenzraumgeräte erforderlich sind, welche auf der Registerkarte für die Konferenzraumsysteme im [Skype for Business Solutions-Katalog](https://go.microsoft.com/fwlink/?LinkId=615775) (nur in englischer Sprache verfügbar) aufgeführt sind.  <br/><br/>  **Aktivieren von Audiokonferenzen**  <br/>  Organisationseinrichtung der Standardeinstellungen für Konferenzbrücke  <br/>  Zuweisen der Konferenzbrücke zu lizenzierten Benutzern  <br/><br/>  **Anleitung zum Aktivieren von Telefonsystem und Anrufplänen (nur USA – nicht verfügbar in GCC High- oder DoD-Plänen)**  <br/>  Organisationseinrichtung der Standardeinstellungen für Cloud Voice  <br/>  Zuweisung von Nummern zu lizenzierten Benutzern  <br/>  Anweisung zum Portieren lokaler Rufnummern über die Benutzeroberfläche bis 999  <br/>  SR-Unterstützung für das Portieren von lokalen Nummern über 999 hinaus  <br/><br/>  **Aktivieren Skype for Business Anleitung zum Onboarding von Livekonferenzen (nicht verfügbar in GCC High- oder DoD-Plänen)**  <br/>  Organisationseinrichtung für den Verbund mit dem Livekonferenzdienst   |
|Microsoft Teams-Onboarding  | Identitäten, die in Azure Active Directory (Azure AD) für Office 365 aktiviert sind.  <br/>  Aktivierte Benutzer für SharePoint Online.  <br/>  Exchange Postfächer vorhanden sind (online und lokal in einer Exchange Hybridkonfiguration).  <br/>  Für Office 365-Gruppen aktiviert.<br/>  <br/> **Hinweis:** *FastTrack unterstützt den Bereich basierend auf der Verfügbarkeit der Features in jeder jeweiligen Umgebung. Teams unterstützt SharePoint lokal nicht.*     <br/> <br/>       **Note***The ideal state is for all users to have their mailboxes on Exchange Online. Benutzer mit lokalen Postfächern müssen ihre Identitäten über Azure AD Verbinden mit dem Office 365 Verzeichnis synchronisiert haben. Wenn sich das Postfach des Benutzers lokal befindet, kann der Benutzer für diese Exchange Hybridkunden keine Connectors hinzufügen oder konfigurieren. | Dienst-Onboarding, einschließlich:  <br/>  *<br/>Exchange Online SharePoint Online <br/> OneDrive for Business Skype for Business Online Microsoft Teams Power BI Project Online Yammer <br/> <br/> <br/> <br/> <br/> <br/> Microsoft 365 Apps                              <br/>*   |Die Online-Clientsoftware wie Project für Office 365, Outlook-Client, OneDrive for Business-Synchronisierungsclient, Power BI Desktop und Skype for Business muss mindestens in der in den [Systemanforderungen für Office](https://go.microsoft.com/fwlink/?LinkID=723597) definierten Version vorliegen.  <br/> Die Installationsprogramme für die Windows- und Mac-Desktop-Clients von Microsoft Teams können unter [https://go.microsoft.com/fwlink/?linkid=839411](https://go.microsoft.com/fwlink/?linkid=839411) heruntergeladen werden.   |
